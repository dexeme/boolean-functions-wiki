We will study the algorithms for determining Linear, Affine, Extended Affine and CCZ equivalence between vectorial boolean functions. Linear Equivalence Given two vectorial boolean functions {\displaystyle F} and {\displaystyle G } we want to determine if there exist Linear permutations {\displaystyle A_1} and {\displaystyle A_2 } such that {\displaystyle F = A_2 \circ G \circ A_1 } . The to and from algorithm This algorithm was presented at eurocrypt 2003 . This algorithm is mainly intended for when the boolean functions are permutation, and we will start by assuming {\displaystyle F } and {\displaystyle G } are permutations. The idea of the algorithm is to go use information gathered about {\displaystyle A_1 } to deduce information about {\displaystyle A_2 } and the other way around. To see how this can work let's say we know some value of {\displaystyle A_1 } , lets say {\displaystyle A_1(x) = y } . We of course also know the value of {\displaystyle G} at {\displaystyle y} so lets say that {\displaystyle G(y) = z } . Then we know that {\displaystyle F(x) = A_2 \circ G \circ A_1(x) = A_2 \circ G(y) = A_2(z) } . So we now know that {\displaystyle A_2(z) } must be equal to {\displaystyle G(y) } . For the other way around let's say we know some value of {\displaystyle A_2 } , lets say {\displaystyle A_2(x) = y } . We know the value of {\displaystyle F^{-1} } at y, lets say {\displaystyle F^{-1}(y) = z } . Then we know that {\displaystyle F(z) = y = A_2 \circ G \circ A_1(z) } so we see that {\displaystyle A_2 \circ G \circ A_1(z) = y } . Since we know that {\displaystyle A_2(x) = y } we need {\displaystyle G \circ A_1(z) = x } , which must mean that {\displaystyle A_1(z) = G^{-1}(y)} . So we now showed how we can deduce information knowing either a value of {\displaystyle A_1 } or {\displaystyle A_2} . Now lets say we now the values of {\displaystyle A_1 } at a set of points. Since {\displaystyle A_1 } is linear we also know it's value at any linear combinations of these points so we can just assume that we know the values at {\displaystyle A_1} at {\displaystyle k} linear independent points, which means that we know the value of {\displaystyle A_1} at {\displaystyle 2^k} points. If we now somehow gain value of a point of {\displaystyle A_1} which is not in the span of the already known points then we can deduce the value of {\displaystyle A_1 } at {\displaystyle 2^k } new points using any linear combination of points including this new point. Then we can use all of these new points and try to deduce points of {\displaystyle A_2} as explained before. We will now explain the complete algorithm before giving the psudocode. Given two permutations {\displaystyle F,G : F_2^n \rightarrow F_2^m } we construct the linear permutations {\displaystyle A_1,A_2} . The algorithm is a backtracking algorithm, and whenever we discover a contradiction we backtrack to the last guess. We first guess two values of {\displaystyle A_1(x) } . Since we now know two values of {\displaystyle A_1 } we can two values of {\displaystyle A_2} , which means we can deduce a third value by linearity. Using this third value we can deduce a value of {\displaystyle A_1} , if this value is not in the span of the already known values of {\displaystyle A_1 } we can deduce two more values of {\displaystyle A_1 } and use this to deduce values of {\displaystyle A_1} and so on. If we ever run out of values before we have finished we will have to make additional guesses. If we ever encounter that a situation where we deduce a value of {\displaystyle A_1 } or {\displaystyle A_2 } , but we have already set them to be something else, we must backtrack to the last guess. Runtime It can be hard to estimate the runtime of this algorithm as it is hard to know how many guesses we have to make. Initially we will have to make two guesses (or just 1 if the s-boxes do not map 0 to 0) to get the algorithm started. Assuming we do not have to make any more guesses the algorithm runs in time {\displaystyle O(n^32^{2n})} ( {\displaystyle O(n^32^n)} if the s-boxes do not map 0 to 0). This assumption seems to hold for random functions, but there are bad cases for example when the functions differ in very few points. In general it seems hard to prove any good runtime guarantee for this algorithm. Affine Equivalence Given vectorial boolean functions {\displaystyle F,G : F_2^n \rightarrow F_2^m } find affine permutations {\displaystyle A_1,A_2} such that {\displaystyle F = A_2 \circ G \circ A_1 } . We can also write this as {\displaystyle F \circ A_1^{-1} = A_2 \circ G } . If {\displaystyle A_1(x) = L_1(x) +a_1 , A_2(x) = L_2(x)+a_2 } then {\displaystyle F(x+a_1)} is linear equivalent with {\displaystyle G(x) + a_2} . So we can guess any affine constants {\displaystyle a_1,a_2} and check whether or not {\displaystyle F(x+a_1)} is linear equivalent with {\displaystyle G(x)+a_2} using any linear equivalence algorithm. This will add a multiplicative factor of {\displaystyle 2^{2n}} to the runtime, but will give us an affine equivalence algorithm. The to and from algorithm (Affine) We can adapt the To and from algorithm to the affine case and only add a multiplicative factor of {\displaystyle 2^n} to the runtime. Instead of comparing {\displaystyle F(x+a_1)} to {\displaystyle G(x)+a_2} for every possible {\displaystyle a_1,a_2} we will instead find a representative function for {\displaystyle F(x+a)} for every {\displaystyle a} and then a representative function for {\displaystyle G(x) + a} for every possible {\displaystyle a} . We will then compare to see if any of these representative functions are equal. The representative for a function is the lexicographic smallest linear equivalent function. To see why this work assume {\displaystyle F,G} are affine equivalent with {\displaystyle F = A_1 \circ G \circ A_2 } where {\displaystyle A_1 = L_1 + a_1} and {\displaystyle A_2 = L_2+a_2 } . Then the functions {\displaystyle F(x+a_1)} will be linear equivalent with {\displaystyle G(x)+a_2} . If we have found the minimal linear representative {\displaystyle F'} of {\displaystyle F(x+a_1)} then since {\displaystyle G(x)+a_2} is linear equivalent with {\displaystyle F} it is also linear equivalent with {\displaystyle F'} so the minimal linear representative of {\displaystyle G(x)+a_2} is at least smaller than {\displaystyle F'} . Using this argument the other way around we get that their linear representatives have to be the same function. To actually compute the minimal representative of a function {\displaystyle F} we do the following. We want to construct {\displaystyle F'} , the minimal permutation which is linear equivalent with {\displaystyle F} . We start by guessing the value of {\displaystyle A_1} at the smallest element of {\displaystyle F_2^n} . Let say {\displaystyle A_1(x) = y} . We now do the same as before with going back and forth between {\displaystyle A_1 } and {\displaystyle A_2} , the only difference we always pick the lowest possible value of {\displaystyle A_1} to deduce a value of {\displaystyle A_2} and vise versa. Also whenever we need the value of a undefined point of {\displaystyle F'} , we simply set to it to the lowest available. Rank Algorithm This algorithm is efficient also for non-permutations but only functions of high algebraic degrees. Rank table The algorithm is based on using the rank table of a boolean functions, which we will now introduce. Given a boolean function {\displaystyle F: F_2^n \rightarrow F_2 } . We are going to consider this object algebraically using the ANF (algebraic normal form). {\displaystyle F = \sum_{u \in F_2^n }\alpha_ux^u } . We can look at {\displaystyle F} as a vector spanned by all monomials {\displaystyle x^{u}=x_{1}^{u_{1}}...x_{n}^{u_{n}}} . Let {\displaystyle F_{\geq d}} be the polynomial containing all monomials of {\displaystyle F} with degree at least {\displaystyle d} . Now given a vectorial boolean functions {\displaystyle F=(F_1,...,F_m)} we define the symbolic rank of {\displaystyle F} as the rank of the vectors {\displaystyle \{F_i\}} (where we view {\displaystyle F_i} as a vector). Denote this as {\displaystyle SR(F)} . We compose functions symbolically as {\displaystyle F \circ A_1 = \sum \alpha_u \cdot (M_u \circ A_1 ) } . We have that {\displaystyle deg(F \circ A_1) \leq deg(F) } . We can also compose {\displaystyle A_2 \circ F } where we replace {\displaystyle x_i} in {\displaystyle A_2} by {\displaystyle F_i} . Let {\displaystyle A : F_2^{n-1} \rightarrow F_2^n } be an affine transformation with {\displaystyle A(x) = L(x) + a} . The range of {\displaystyle A} is an affine {\displaystyle n-1} dimensional subspace so it's orthogonal subspace is 1 dimensional, so spanned by a single vector {\displaystyle h} . We call {\displaystyle h} the half space mask (HSM), since it partitions the space into 2 halves. {\displaystyle h \cdot a } is the half space free coefficient (HSC). Given an HSM and HSC {\displaystyle h} and {\displaystyle c} there is a canonical affine transformation {\displaystyle C_{|_{h,c}} : F_2^{n-1} \rightarrow F_2^n } . We can now define the rank table of {\displaystyle F} with respect to some constant {\displaystyle d} . For any {\displaystyle h \in F_2^n } we calculate {\displaystyle u = SR((F \circ C_{|_{h,0}})_{\geq d} )} and {\displaystyle v = SR((C_{|_{h,1}})_{\geq d}) } . The rank table entry for {\displaystyle h} then becomes {\displaystyle (max(u,v),min(u,v))} . For any specific tuple {\displaystyle (u,v)} the rank group is all {\displaystyle h} such that the rank table entry for {\displaystyle h} is {\displaystyle (u,v)} . The rank histogram is a mapping for each {\displaystyle (u,v)} to the size of the rank group. One last thing we will need is the concept of the rank histogram with with respect to a given rank group. Fix an element {\displaystyle h \in F_2^n } . The rank histogram of {\displaystyle h} with respect to the rank group {\displaystyle (u,v)} is defined the following way. Add {\displaystyle h} to all elements of the rank group {\displaystyle (u,v)} to get a set {\displaystyle U \subset F_2^n } . For each element {\displaystyle h'} of {\displaystyle U} we look at the rank group containing {\displaystyle h'} , let's say it's {\displaystyle (u',v')} . The multi set containing the tuples {\displaystyle (u',v')} for each element {\displaystyle h' \in U } is the rank histogram of {\displaystyle h} with respect to {\displaystyle (u,v)} . The rank group {\displaystyle (u,v) } with respect to {\displaystyle (u',v')} is the multiset rank histogram of each element {\displaystyle h} of the group {\displaystyle (u,v)} with respect to the group {\displaystyle (u',v')} . Algorithm The main idea of the algorithm is that if {\displaystyle F} and {\displaystyle G} are affine equivalent, then {\displaystyle SR(F_{\geq d}) = SR(G_{\geq d}) } . If {\displaystyle F} and {\displaystyle G} are affine equivalent with {\displaystyle F = A_2 \circ G \circ A_1 } we are going to start by trying to reconstruct {\displaystyle A_1} . Do do this the idea is that for any element of the rank table of {\displaystyle (u,v)} of {\displaystyle G } . So if we find a entry {\displaystyle (u,v) } which contains only 1 element we know one point of {\displaystyle A_1} . Now the rank groups can be large so only doing this may still result in a lot of guesses. For some group {\displaystyle (u,v)} we are going to pick another group {\displaystyle (u',v')} and calculate the rank group of {\displaystyle (u,v)} with respect to {\displaystyle (u',v')} . If this multiset contains an unique element, then the element {\displaystyle h} of {\displaystyle (u,v)} corresponding to this element must be matched to the corresponding element of {\displaystyle G} (Due to the linearity of {\displaystyle A_1} ). We can do this for any pair of groups, and if we find any element in the multiset of low cardinality we obtain a lot of information about possible matchings for {\displaystyle A_1} . Runtime The algorithm is estimated to be {\displaystyle O(n^32^n)} for a random permutation. This is based to some assumption about the distribution of the rank tables of random functions. EA equivalence Given two boolean functions {\displaystyle F,G : F_2^n \rightarrow F_2^m } find two affine permutations {\displaystyle A_1,A_2} and an affine transformation {\displaystyle A_3} such that {\displaystyle F = A_2 \circ G \circ A_1 + A_3 } Jacobian algorithm This algorithm can decide EA equivalence for quadratic functions only. The Jacobian Given a vectorial boolean function, and any element {\displaystyle a \in F_2^n} the deriviative in direction {\displaystyle a} is defined by {\displaystyle D_a F(x) = F(x+a) + F(x) } . The Jacobian for a vectorial boolean function {\displaystyle F(x) = (F_1(x),...,F_m(x)) } is defined as {\displaystyle J F(x) = \begin{pmatrix}D_{e_1}F_1(x) & D_{e_2}F_1(x) & ... & D_{e_n}F_1(x) \\ D_{e_1}F_2(x) & D_{e_2}F_2(x) & ... & D_{e_n}F_2(x) \\ \vdots & \vdots & \vdots & \vdots \\ D_{e_1}F_m(x) & D_{e_2}F_m(x) & ... & D_{e_n}F_m(x) \end{pmatrix} } where {\displaystyle e_i} is the i-th basis vector of {\displaystyle F_2^n} . We denote the linear part of the jacobian by {\displaystyle J_{lin}F(x)} . The algorithm The algorithm is based on the following two facts that if {\displaystyle F} and {\displaystyle G} are EA equivalent quadratic functions with {\displaystyle F = A_2 \circ G \circ A_1 + A_3 } we can assume that {\displaystyle A_1} and {\displaystyle A_3} are linear. So we have just {\displaystyle A_2(x) = L_2(x) + a_2 } . The other fact is that {\displaystyle J_{lin}F(x) = L_2\cdot J_{lin}G(A_1(x)) \cdot A_1 } . This allows the us to start by searching for pairs {\displaystyle (L_2,A_1) } first, and deduce the other values later. To deduce possible pairs {\displaystyle (L_2,A_1)} the algorithm does the following. We are first gonna try to find {\displaystyle A_1} . Using the fact that the rank of the matrix {\displaystyle Jac_{lin}F(x) } equals the rank of {\displaystyle Jac_{lin}G(A_1(x)) } since all of the matrices/transformations are permutations. This means that {\displaystyle A_1(x)} can only be mapped to elements which results in the rank being the same. So we are going to compute all possible ranks of {\displaystyle Jac_{lin}F(x)} and {\displaystyle Jac_{lin}G(x)} . We are then going to look at the least common rank of these tables, let say this value is {\displaystyle k} . Let {\displaystyle S_F} be all inputs such that {\displaystyle Jac_{lin} F(x) } has rank {\displaystyle k} and {\displaystyle S_G} all {\displaystyle x} such that {\displaystyle Jac_{lin}G(x)} has rank {\displaystyle k} . We then know by the previous observation that {\displaystyle A_1} has to map elements from {\displaystyle S_F} to elements of {\displaystyle S_G} . If these sets {\displaystyle S_F} and {\displaystyle S_G} are small (we can assume they are the same size) then the number of guesses we have will not be to large. To start with we will guess the value of {\displaystyle A_1} of some elements of {\displaystyle S_F} . Having guessed some values of {\displaystyle A_1} we are going to deduce values of {\displaystyle L_2} . If we have guessed that {\displaystyle A_1u = w} . Then the pair {\displaystyle (L_2,A_1) = (X,Y)} is a solution to the linear system of equations {\displaystyle X\cdot Jac_{lin}F(v) - Jac_{lin}F(w) \cdot Y = 0 } {\displaystyle Y \cdot v = 0 } We are going to guess enough values of {\displaystyle A_1} so that this system has an unique solution (Since each guess gives us more equations). Having done this and found a pair {\displaystyle (L_2,A_1)} we can deduce {\displaystyle A_3} and {\displaystyle a_2} with basic linear algebra. Lets now describe the algorithm in more detail. 1. Compute the rank table of {\displaystyle F} . {\displaystyle R(F) = \{x \in F_2^n | rank(Jac_{lin}F(x)) = j \} } . Do the same for {\displaystyle G} 2. Let {\displaystyle s} be the number of guesses of {\displaystyle A_1} we are going to make. Let {\displaystyle i = \min |R(F) | } . Pick {\displaystyle s} elements of {\displaystyle R(F) } . We are then gonna guess all possible values of {\displaystyle A_1} on these points. But we only have to guess values inside {\displaystyle R } . Let's say we made the guesses {\displaystyle A_1u_1 = w_1,...,A_1u_s = w_s} . 3. Try to solve the {\displaystyle s} system of equations {\displaystyle X \cdot Jac_{lin}F(v_i) - Jac_{lin}F(w_i)\cdot Y,\: Y \cdot v_i = 0 } . If this system has to many solutions make another guess (increase the value of {\displaystyle s} temporary). 4. When the system of equations does not have to many solutions find all solutions {\displaystyle (L_2,A_1)} . Then deduce the rest of the values {\displaystyle A_3} , {\displaystyle a_2} . If this is possible we are done, otherwise go back to step 2 and make another guess. Runtime The runtime of this algorithm is related to the rank table which is related to the differential uniformity of the function. Let {\displaystyle R = \min R(F) } . Then we will at worst have to make around {\displaystyle R^s} guesses. For each such guess we will have to solve some linear equations which can be done in around {\displaystyle (n^2+m^2)^w} where {\displaystyle w} is a matrix multiplication constant. In total we get a time of {\displaystyle O(max(n,m)^w2^n + R^s(m^2+n^2)^w)} , where the first part is for computing the rank table. Note that when {\displaystyle F} is APN all the values have the same rank, so {\displaystyle R = 2^n} . Which is the worst case for this algorithm.
